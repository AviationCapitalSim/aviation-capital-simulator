# ACS TECHNICAL SESSION LOG
ID: TSL-2026-02-26-A
Módulo: Used Market / My Aircraft — Pending Rule Correction
Estado: CERRADO

---

## 1. Problema Detectado

❌ El 4to avión comprado desde Used Market NO entraba en estado `Pending`.

Síntomas observados:

- Todos los aviones comprados quedaban con `status: "Active"`.
- La regla estructural de “3 activos / resto Pending” no se estaba respetando.
- El comportamiento era inconsistente con la lógica histórica definida para la flota.
- No existía control dinámico basado en tamaño actual de `ACS_MyAircraft`.

Esto rompía la coherencia operativa del sistema de control de flota.

---

## 2. Diagnóstico Técnico

El problema NO estaba en:

- Registration Manager
- Time Engine
- My Aircraft renderer
- Finance Engine

El problema se encontraba en la función `buyUsed()` dentro de `acs_used_market.js`.

La inserción directa de aeronave contenía:

status: "Active",

Esto forzaba todas las aeronaves usadas a estado Active, anulando completamente la regla estructural.

No existía evaluación del tamaño actual de la flota antes de asignar el estado.

---

## 3. Arquitectura Respetada

Se decidió mantener la estructura canonical actual:

- No usar `createFleetAircraft()`
- No modificar flujo financiero
- No alterar sistema de matrícula
- No alterar preservación de hours / cycles
- No modificar estructura UA-NEW-STRUCT-1

Se mantuvo la inserción directa en `ACS_MyAircraft`.

La corrección debía ser mínima, aislada y sin impacto colateral.

---

## 4. Corrección Implementada

Se agregó evaluación dinámica del tamaño actual de flota antes de crear el objeto:

let myFleet = JSON.parse(localStorage.getItem("ACS_MyAircraft") || "[]");

// Regla 3 activos / resto Pending
let status = myFleet.length < 3 ? "Active" : "Pending";

Y dentro del objeto se reemplazó:

status: status,

Eliminando el hardcode:

status: "Active",
