## 1. Objetivo

Implementar un modal informativo de aeronave (Aircraft Dossier) dentro de `My Aircraft`, con diseño Qatar Luxury + Airbus OCC, independiente del sistema legacy de modales y compatible con arquitectura multiplayer (700+ jugadores).

El modal debe mostrar:

- Imagen
- Manufacturer + Model
- Registration
- Seats (desde DB real)
- Range
- Engines
- Year
- Purchase Value
- Market Value
- Liquidity Index

---

## 2. Problemas Detectados

### ❌ 2.1 Conflicto de IDs entre HTML y JS

El JS intentaba escribir en elementos inexistentes:

- infoModalTitle
- infoModalRegistration
- infoModalImage

Error generado en consola:

TypeError: null is not an object

Causa:
Los IDs definidos en el JS no coincidían con los IDs reales del HTML.

---

### ❌ 2.2 Mezcla de Sistemas de Modal

Existían múltiples funciones activas:

- openAircraftModal()
- openAircraftInfoModal()
- openModelAircraftModal()

Esto generó:

- Llamadas a funciones no existentes
- Referencias a closeAircraftInfoModal()
- Ruptura parcial del script en Safari

Causa:
No se aisló completamente el nuevo modal del sistema legacy.

---

### ❌ 2.3 Reutilización Incorrecta de Namespace

Se intentó reutilizar prefijos del modal antiguo (infoModal*).

Esto generó:

- Confusión estructural
- Colisiones semánticas
- Errores de mantenimiento futuro

---

### ❌ 2.4 Overlay Mal Aislado

Se intentó usar clase `.modal` global.

Riesgo:
Romper otros modales existentes (maintenance, aircraft view, etc.).

Conclusión:
Cada modal debe tener overlay independiente por ID.

---

### ❌ 2.5 Ruta de Imagen Incorrecta

Se utilizó:

images/manufacturer_model.jpg

Pero la estructura real del proyecto es:

img/Manufacturer/model.png|jpg

Resultado:
Imagen no cargaba correctamente.

---

## 3. Diagnóstico Arquitectónico

El problema NO fue del Time Engine.
NO fue de Safari.
NO fue del render de la tabla.

El problema fue:

Inconsistencia estructural entre HTML y JS + falta de namespace exclusivo para el nuevo modal.

Principio que no se respetó inicialmente:

- Un modal = un contenedor único
- IDs exclusivos
- Namespace propio
- Sin reutilizar identificadores legacy

---

## 4. Corrección Aplicada

### ✅ 4.1 Creación de Modal Independiente

Nuevo contenedor exclusivo:

#aircraftInfoModal

Con IDs normalizados:

- dossierTitle
- dossierRegistration
- dossierImage
- dossierSeats
- dossierRange
- dossierEngines
- dossierYear
- dossierPurchase
- dossierMarket
- dossierSell

Sin reutilizar infoModal*.

---

### ✅ 4.2 Aislamiento Total del Sistema Legacy

No se modificaron:

- aircraftModal
- maintenanceLogModal
- buy_new modal
- used market modal

El Dossier ahora es independiente.

---

### ✅ 4.3 Namespace Limpio

Todos los elementos del nuevo modal usan prefijo:

dossier*

Eliminando colisión con sistemas anteriores.

---

### ✅ 4.4 Implementación Financiera Inicial

Se implementó cálculo base:

marketValue = purchase * (1 - depreciation * age)
sellValue = marketValue * 0.9

Usando año del simulador (getSimTime).

Pendiente refinar modelo histórico realista por era.

## 6. Lección Técnica

Un modal aparentemente simple puede volverse inestable si:

- Se reutilizan IDs
- Se mezclan sistemas legacy
- No se define namespace propio
- No se respeta aislamiento modular

La solución correcta no fue parchear,
sino aislar completamente el nuevo sistema.
